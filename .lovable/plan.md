
# P2P Marketplace Upgrade

## 1. Backend — Lovable Cloud

Enable Lovable Cloud and add `STEAM_WEB_API_KEY` as a runtime secret.

**Schema (migrations):**
- `profiles` — `id (uuid, FK auth.users)`, `steam_id (text, unique)`, `persona`, `avatar_url`, `profile_url`, `last_inventory_sync`, `created_at`
- `listings` — `id`, `seller_id (FK auth.users)`, `steam_id`, `asset_id`, `market_hash_name`, `weapon`, `name`, `wear`, `float`, `pattern`, `stickers (jsonb)`, `inspect_link`, `icon_url`, `price_usd (numeric)`, `status (enum: active|sold|unavailable|removed)`, `created_at`, `updated_at`, `last_validated_at`
  - Unique partial index on `(steam_id, asset_id)` where `status='active'` (prevents duplicates)
- `inventory_cache` — `steam_id (pk)`, `items (jsonb)`, `fetched_at` (15-min TTL)

**RLS:**
- `profiles`: read public, update own
- `listings`: read all (active/sold/unavailable visible to everyone), insert/update/delete own (`seller_id = auth.uid()`)
- `inventory_cache`: server-only (service role)

## 2. Steam OpenID 2.0 Login

Server routes under `src/routes/api/public/steam/`:
- `login.ts` — redirects to `https://steamcommunity.com/openid/login` with proper params (return_to, realm)
- `callback.ts` — verifies the OpenID response with Steam (`check_authentication`), extracts SteamID64, fetches profile via `ISteamUser/GetPlayerSummaries`, upserts into `profiles`, creates a Supabase session for the user (using admin `signInWithOtp` magic-link bypass: generate a deterministic email like `{steamid}@steam.fragmarket` and use `supabaseAdmin.auth.admin.generateLink` to obtain a session, then redirect with hash tokens).

Login button in `TopBar` opens `/api/public/steam/login`.

## 3. Steam Inventory

Server function `getMyInventory` (auth-protected):
- Reads cache → if fresh (<15 min) return cached
- Otherwise fetches `https://steamcommunity.com/inventory/{steamid}/730/2?l=english&count=5000`
- Parses `assets` + `descriptions` into a flat list `{ asset_id, market_hash_name, icon_url, name, weapon, wear, tradable }`
- Caches in `inventory_cache`

## 4. Listings — Server Functions

All in `src/lib/listings.functions.ts`:
- `createListing({ asset_id, price_usd, inspect_link })` — verifies the asset exists in seller's *current* Steam inventory; mocks float/pattern/stickers from `inspect_link` (deterministic hash); inserts row
- `updateListingPrice({ id, price_usd })` — own listings only
- `removeListing({ id })` — sets status `removed`
- `relistListing({ id, price_usd })` — re-validates ownership, re-activates
- `revalidateListings({ steamId })` — fetches inventory, marks listings whose asset_id is missing as `unavailable` (called from cron-like client poll + on profile open)

## 5. UI

**Keep existing design tokens / cards.** Add:
- **Login button** in TopBar (`Sign in with Steam` with Steam logo), shows avatar/persona once logged in (replaces wallet placeholder)
- **`/inventory`** — auth-guarded page showing real Steam inventory grid; each item has a "List for sale" dialog (price input, inspect link auto-filled)

- **Marketplace** (`/marketplace`) — replace mock `SKINS` with live `useQuery` of active listings; keep existing filters; add **sticker search** (text contains in stickers array), **float sort**, **price sort**, **exterior filters** (wear chips already exist); add `LISTED` / `UNAVAILABLE` / `SOLD` badges on `SkinCard`
- Lazy load via `IntersectionObserver`-based pagination (50 per page)

## 6. Realtime

Subscribe to `listings` table inserts/updates on `/marketplace` via `supabase.channel('listings')` → invalidates the React Query cache so new listings/price changes appear instantly.

## 7. Validation Jobs

- `useEffect` interval on `/marketplace` triggers `revalidateListings` for the top N visible sellers every 5 min (batched, server-side cached so multiple users don't hammer Steam)
- On `/my-listings` mount, immediately revalidate the current user's listings

## Technical Notes

- Steam inventory endpoint can be slow / rate-limited; the 15-min `inventory_cache` and per-steamid mutex (in-memory map on server) prevent stampedes
- Inspect link mock: `float = (hash(asset_id) % 1000) / 1000 * 0.8`, `pattern = hash % 1000`
- No payments / no trade automation — buy CTA simply opens Steam profile chat URL (`https://steamcommunity.com/profiles/{steamid}`)
- All Steam image URLs already use `community.cloudflare.steamstatic.com` from previous turn
